# 12장 함수

## 12.1 함수란?

- **함수**: 일련의 과정을 문(statement)으로 구현하고 코드 블록으로 감싸서 하나의 실행 단위로 정의한 것
- 수학의 함수와 같은 개념: 입력(input)을 받아 출력(output)을 반환

```javascript
// 함수 정의
function add(x, y) {
  return x + y;
}

// 함수 호출
var result = add(2, 5);
console.log(result); // 7
```

| 용어                      | 설명                             |
| ------------------------- | -------------------------------- |
| **매개변수 (parameter)**  | 함수 내부로 입력을 전달받는 변수 |
| **인수 (argument)**       | 함수 호출 시 전달하는 입력 값    |
| **반환값 (return value)** | 함수의 출력                      |

## 12.2 함수를 사용하는 이유

1. **코드 재사용**: 실행 시점을 개발자가 결정하고 몇 번이든 재사용 가능
2. **유지보수 편의성**: 코드 중복 제거로 수정이 용이
3. **코드 신뢰성**: 실수를 줄여 코드의 신뢰성 향상
4. **코드 가독성**: 적절한 함수 이름으로 내부 코드 없이도 역할 파악 가능

> **가독성이 좋은 코드가 좋은 코드다**

## 12.3 함수 리터럴

자바스크립트의 함수는 **객체 타입의 값**이다. 함수도 리터럴로 생성할 수 있다.

```javascript
var f = function add(x, y) {
  return x + y; // 변수에 함수 리터럴 할당 가능
};
```

**함수 리터럴 구성 요소**

| 구성 요소         | 설명                                                                                 |
| ----------------- | ------------------------------------------------------------------------------------ |
| **함수 이름**     | 식별자 네이밍 규칙 준수, 함수 몸체 내에서만 참조 가능, 생략 가능 (기명함수/익명함수) |
| **매개변수 목록** | 0개 이상을 소괄호로 감싸고 쉼표로 구분, 순서에 의미 있음, 식별자 네이밍 규칙 준수    |
| **함수 몸체**     | 함수 호출 시 일괄 실행될 문들을 하나의 실행 단위로 정의한 코드 블록                  |

<small>※ **리터럴**: 사람이 이해할 수 있는 문자 또는 약속된 기호를 사용해 값을 생성하는 표기 방식</small>

> 함수 리터럴도 평가되어 값을 생성하며, 이 값은 객체다. 즉, **함수는 객체다**

**일반 객체 vs 함수 객체**

| 구분          | 일반 객체 | 함수 객체                                            |
| ------------- | --------- | ---------------------------------------------------- |
| 호출          | 불가능    | **가능**                                             |
| 고유 프로퍼티 | 없음      | 있음 (arguments, caller, length, name, prototype 등) |

## 12.4 함수 정의

**함수를 정의하는 4가지 방법**

| 방식                 | 예시                                                |
| -------------------- | --------------------------------------------------- |
| 함수 선언문          | `function add(x, y) { return x + y; }`              |
| 함수 표현식          | `var add = function(x, y) { return x + y; };`       |
| Function 생성자 함수 | `var add = new Function('x', 'y', 'return x + y');` |
| 화살표 함수 (ES6)    | `const add = (x, y) => x + y;`                      |

> **선언 vs 정의**: 변수는 "선언"한다고 하고, 함수는 "정의"한다고 표현 (ECMAScript 사양)

### 12.4.1 함수 선언문

```javascript
function add(x, y) {
  return x + y;
}

console.log(add(2, 5));  // 7

// 함수 이름 생략 불가
function (x, y) { return x + y; }  // SyntaxError
```

- 함수 선언문은 **표현식이 아닌 문**
- 하지만 변수에 할당하는 것처럼 보이는 경우가 있음

```javascript
var add = function add(x, y) {
  return x + y;
};
```

**중의적인 코드**

JS 엔진은 코드의 문맥에 따라 동일한 코드를 다르게 해석:

```javascript
// {} - 단독 사용 → 블록문
{
}

// {} - 값으로 평가되어야 할 문맥 → 객체 리터럴
var obj = {};
```

```javascript
// 단독 사용 → 함수 선언문으로 해석 (이름 생략 불가)
function foo() {
  console.log("foo");
}
foo(); // 'foo'

// 그룹 연산자 내부 → 함수 리터럴 표현식으로 해석
(function bar() {
  console.log("bar");
});
bar(); // ReferenceError: bar is not defined
```

**왜 `bar`는 에러가 발생하는가?**

- 함수 리터럴 표현식으로 해석되면 함수 이름은 함수 몸체 내에서만 참조 가능
- 외부에서 `bar`라는 식별자로 접근할 수 없음

**왜 `foo`는 호출이 가능한가?**

- 함수 선언문으로 해석되면 JS 엔진이 **함수 이름과 동일한 식별자를 암묵적으로 생성**
- 그 식별자에 함수 객체를 할당

```javascript
// 실제로는 이렇게 동작
var add = function add(x, y) {
  return x + y;
};
//  ↑ 식별자   ↑ 함수 이름

console.log(add(2, 5)); // 함수 이름이 아닌 식별자로 호출
```

### 12.4.2 함수 표현식

**일급 객체**: 값의 성질을 갖는 객체. JS의 함수는 일급 객체이므로 값처럼 자유롭게 사용 가능

```javascript
// 익명 함수 표현식 (일반적)
var add = function (x, y) {
  return x + y;
};

// 기명 함수 표현식
var add = function foo(x, y) {
  return x + y;
};
console.log(foo(2, 5)); // ReferenceError (함수 이름은 함수 몸체 내에서만 유효)
```

| 구분      | 함수 선언문      | 함수 표현식 |
| --------- | ---------------- | ----------- |
| 문의 종류 | 표현식이 아닌 문 | 표현식인 문 |

### 12.4.3 함수 생성 시점과 함수 호이스팅

```javascript
console.dir(add); // ƒ add(x, y)
console.dir(sub); // undefined
console.log(add(2, 5)); // 7
console.log(sub(2, 5)); // TypeError: sub is not a function

function add(x, y) {
  return x + y;
} // 함수 선언문
var sub = function (x, y) {
  return x - y;
}; // 함수 표현식
```

**함수 호이스팅 vs 변수 호이스팅**

| 구분         | 함수 선언문                   | 함수 표현식 (var)               |
| ------------ | ----------------------------- | ------------------------------- |
| 호이스팅     | 함수 호이스팅                 | 변수 호이스팅                   |
| 선언 전 호출 | **가능** (함수 객체로 초기화) | **불가능** (undefined로 초기화) |

> **함수 호이스팅**: 함수 선언문이 코드의 선두로 끌어 올려진 것처럼 동작하는 JS 특징

### 12.4.4 Function 생성자 함수

```javascript
var add = new Function("x", "y", "return x + y");
console.log(add(2, 5)); // 7
```

- `new` 없이도 사용 가능
- **권장하지 않음**: 클로저를 생성하지 않는 등 다르게 동작

### 12.4.5 화살표 함수

ES6에서 도입. `function` 키워드 대신 `=>`를 사용해 간략하게 정의

```javascript
const add = (x, y) => x + y;
console.log(add(2, 5)); // 7
```

**화살표 함수의 특징**

| 특징        | 설명                             |
| ----------- | -------------------------------- |
| 익명 함수   | 항상 익명 함수로 정의            |
| 생성자 함수 | 사용 불가                        |
| this 바인딩 | 상위 스코프의 this를 그대로 참조 |
| prototype   | 프로퍼티 없음                    |
| arguments   | 객체 생성하지 않음               |

> 자세한 내용은 26.3장에서 설명

## 12.5 함수 호출

함수는 함수 호출 연산자 `()`로 호출

### 12.5.1 매개변수와 인수

```javascript
function add(x, y) {
  return x + y;
}

var result = add(1, 2); // 인수 1, 2가 매개변수 x, y에 순서대로 할당
```

- 함수 호출 시 매개변수 생성 → `undefined`로 초기화 → 인수 할당
- 매개변수는 함수 내부에서만 참조 가능 (외부에서 참조 시 ReferenceError)

```javascript
console.log(add(2)); // NaN (y는 undefined)
console.log(add(2, 5, 10)); // 7 (초과 인수는 무시, arguments에 보관)
```

### 12.5.2 인수 확인

```javascript
function add(x, y) {
  return x + y;
}

console.log(add("a", "b")); // 'ab' (의도하지 않은 결과)
```

- JS는 동적 타입 언어라 매개변수 타입을 사전에 지정할 수 없음
- **TypeScript** 사용으로 타입 체크 가능
- 인수 개수는 `arguments` 객체를 통해 확인 가능

```javascript
function add(x, y) {
  if (typeof x !== "number" || typeof y !== "number") {
    throw new TypeError("인수는 모두 숫자여야 합니다.");
  }
  return x + y;
}
```

### 12.5.3 매개변수의 최대 개수

- 이상적인 매개변수 개수는 **0개**, 적을수록 좋음
- 이상적인 함수는 **한 가지 일**만 해야 하며, 가급적 **작게** 만들어야 함
- 3개 이상이면 객체로 전달하는 것이 유리

### 12.5.4 반환문

`return` 키워드와 표현식으로 실행 결과를 함수 외부로 반환

```javascript
function multiply(x, y) {
  return x * y;
}

var result = multiply(3, 5);
console.log(result); // 15
```

**반환문의 역할**

1. 함수 실행을 중단하고 함수 몸체를 빠져나감
2. `return` 뒤의 표현식을 평가해 반환

```javascript
function multiply(x, y) {
  return;
  x * y; // 무시됨 (줄바꿈으로 인해 ASI가 세미콜론 삽입)
}

console.log(multiply(3, 5)); // undefined
```

> 반환문은 함수 내부에서만 사용 가능. 전역에서 사용 시 SyntaxError

## 12.6 참조에 의한 전달과 외부 상태의 변경

매개변수도 변수와 동일하게 **값에 의한 전달** / **참조에 의한 전달** 방식을 따름

```javascript
function changeVal(primitive, obj) {
  primitive += 100;
  obj.name = "Kim";
}

var num = 100;
var person = { name: "Lee" };

console.log(num); // 100
console.log(person); // { name: 'Lee' }

changeVal(num, person);

console.log(num); // 100 (변경 안 됨 - 값에 의한 전달)
console.log(person); // { name: 'Kim' } (변경됨 - 참조에 의한 전달)
```

> **해결책**: 불변 객체로 만들어 사용 (객체를 원시 값처럼 변경 불가능하게)

## 12.7 다양한 함수의 형태

### 12.7.1 즉시 실행 함수 (IIFE)

정의와 동시에 즉시 호출되며, 재호출 불가

```javascript
// 익명 즉시 실행 함수 (일반적)
(function () {
  var a = 3;
  var b = 5;
  return a + b;
})();

// 기명 즉시 실행 함수
(function foo() {
  var a = 3;
  var b = 5;
  return a + b;
})();

foo(); // ReferenceError (외부에서 호출 불가)
```

**`()`로 감싸는 이유**

```javascript
function() { ... }();  // SyntaxError (함수 선언문은 이름 생략 불가)

function foo() { ... }();  // SyntaxError (세미콜론 자동 삽입으로 ()가 그룹 연산자로 해석)
```

- `()`로 감싸면 함수 리터럴로 평가되어 함수 객체 생성

```javascript
// 값 반환 가능
var res = (function () {
  return 10 + 20;
})();
console.log(res); // 30

// 인수 전달 가능
res = (function (a, b) {
  return a * b;
})(3, 5);
console.log(res); // 15
```

### 12.7.2 재귀 함수

자기 자신을 호출하는 함수

```javascript
function countdown(n) {
  if (n < 0) return;
  console.log(n);
  countdown(n - 1); // 재귀 호출
}

countdown(10);
```

**팩토리얼 구현**

```javascript
function factorial(n) {
  if (n <= 1) return 1;
  return n * factorial(n - 1);
}

console.log(factorial(5)); // 120 (5 * 4 * 3 * 2 * 1)
```

> **주의**: 탈출 조건을 반드시 만들어야 함. 없으면 무한 호출 → **스택 오버플로 에러**

### 12.7.3 중첩 함수

함수 내부에 정의된 함수를 **중첩 함수(내부 함수)**, 이를 포함한 함수를 **외부 함수**라 함

```javascript
function outer() {
  var x = 1;

  function inner() {
    var y = 2;
    console.log(x + y); // 외부 함수의 변수 참조 가능
  }

  inner();
}

outer(); // 3
```

- 중첩 함수는 외부 함수 내부에서만 호출 가능
- 외부 함수의 변수를 참조할 수 있음 (클로저와 관련)

### 12.7.4 콜백 함수

| 용어          | 설명                                                      |
| ------------- | --------------------------------------------------------- |
| **콜백 함수** | 함수의 매개변수를 통해 다른 함수의 내부로 전달되는 함수   |
| **고차 함수** | 매개변수를 통해 함수의 외부에서 콜백 함수를 전달받은 함수 |

```javascript
// 고차 함수
function repeat(n, callback) {
  for (var i = 0; i < n; i++) {
    callback(i);
  }
}

// 콜백 함수
var logAll = function (i) {
  console.log(i);
};

var logOdds = function (i) {
  if (i % 2) console.log(i);
};

repeat(5, logAll); // 0 1 2 3 4
repeat(5, logOdds); // 1 3
```

- 고차 함수는 콜백 함수의 호출 시점을 결정하여 호출
- 콜백 함수는 고차 함수에 의해 호출되며, 필요에 따라 인수를 전달받음

### 12.7.5 순수 함수와 비순수 함수

| 구분           | 순수 함수 | 비순수 함수 |
| -------------- | --------- | ----------- |
| 부수 효과      | 없음      | 있음        |
| 외부 상태 의존 | X         | O           |
| 외부 상태 변경 | X         | O           |

**순수 함수**

```javascript
var count = 0;

function increase(n) {
  return ++n; // 외부 상태 변경 없이 새 값 반환
}

count = increase(count);
console.log(count); // 1
```

**비순수 함수**

```javascript
var count = 0;

function increase() {
  return ++count; // 외부 상태(count) 직접 변경
}

increase();
console.log(count); // 1
```

> **함수형 프로그래밍**: 순수 함수와 보조 함수의 조합으로 부수 효과를 최소화하여 불변성 지향
