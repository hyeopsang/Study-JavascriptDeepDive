# 13장 스코프

## 13.1 스코프란?

**스코프의 의미는 유효범위**

자바스크립트에서 `var`로 선언한 변수와 `let`, `const`로 선언한 변수의 스코프가 다르게 동작한다.

예를 들어, 함수의 매개변수는 함수 내부에서만 참조할 수 있고 함수 외부에서는 참조할 수 없다. 이는 매개변수의 유효범위(스코프)가 한정되어 있기 때문이다.

### 핵심 개념

- **모든 식별자는 자신이 선언된 위치에 의해 참조할 수 있는 유효범위가 결정된다** ← 이것이 스코프
- 자바스크립트 엔진은 이름이 같은 두 개의 변수 중에서 어떤 변수를 참조해야 할 것인지 결정하는데, 이를 **식별자 결정**이라고 한다
- 스코프는 식별자를 검색할 때 사용하는 규칙이다

### 관련 용어

- **렉시컬 환경**: 코드가 어디서 실행되며 주변에 어떤 코드가 있는지
- **실행 컨텍스트**: 렉시컬 환경을 구현한 것 (23장에서 자세히 설명)

### 스코프의 역할

스코프를 통해 변수 이름의 충돌을 방지하여 같은 이름의 변수를 사용할 수 있게 한다 (네임스페이스).

**중복 선언 규칙**

- `var`: 같은 스코프 내에서 중복 선언 허용
- `let`, `const`: 같은 스코프 내에서 중복 선언 불허용 ← `let`, `const`를 사용하는 이유

## 13.2 스코프의 종류

### 스코프 구분

| 구분 | 설명                  | 스코프      | 변수      |
| ---- | --------------------- | ----------- | --------- |
| 전역 | 코드의 가장 바깥 영역 | 전역 스코프 | 전역 변수 |
| 지역 | 함수 몸체 내부        | 지역 스코프 | 지역 변수 |

### 13.2.1 전역과 전역 스코프

```javascript
var x = "global x";
var y = "global y";

function outer() {
  var z = "outer's local z";

  console.log(x); // 'global x'
  console.log(y); // 'global y'
  console.log(z); // 'outer\'s local z'

  function inner() {
    var x = "inner's local x";

    console.log(x); // 'inner\'s local x'
    console.log(y); // 'global y'
    console.log(z); // 'outer\'s local z'
  }

  inner();
}

outer();

console.log(x); // 'global x'
console.log(z); // ReferenceError: z is not defined
```

**전역 변수는 어디서든지 참조할 수 있다.**

### 13.2.2 지역과 지역 스코프

- **지역**은 함수 내부를 뜻한다
- 지역 변수는 자신의 지역 스코프와 하위 지역 스코프에서 유효하다
- 위 예제에서 `inner` 함수의 `x`는 `'inner\'s local x'`가 출력된다

## 13.3 스코프 체인

함수가 중첩되듯이 함수의 지역 스코프도 중첩될 수 있다. 스코프는 중첩에 의해 계층적 구조를 갖는다.

### 스코프 계층 구조

```
전역 스코프
  └─ outer 지역 스코프
       └─ inner 지역 스코프
```

| 스코프            | 변수 |
| ----------------- | ---- |
| 전역 스코프       | x, y |
| outer 지역 스코프 | z    |
| inner 지역 스코프 | x    |

### 스코프 체인의 동작 원리

- **스코프 체인**: 스코프가 계층적으로 연결된 것
- 변수를 참조할 때 자바스크립트 엔진은 스코프 체인을 통해 변수를 참조하는 코드의 스코프에서 시작하여 **상위 스코프 방향으로 이동**하며 선언된 변수를 검색한다

자바스크립트 엔진은 코드를 실행하기에 앞서 렉시컬 환경을 실제로 생성한다.

> **스코프 체인**: 실행 컨텍스트의 렉시컬 환경을 단방향으로 연결한 것

### 13.3.1 스코프 체인에 의한 변수 검색

**핵심 규칙**

- 상위 스코프에서 유효한 변수는 하위 스코프에서 자유롭게 참조할 수 있다
- 하위 스코프에서 유효한 변수를 상위 스코프에서 참조할 수 없다

### 13.3.2 스코프 체인에 의한 함수 검색

```javascript
function foo() {
  console.log("global function foo");
}

function bar() {
  function foo() {
    console.log("local function foo");
  }
  foo(); // 'local function foo'
}

bar();
```

함수를 호출하면 암묵적으로 식별자를 가지기 때문에 함수도 스코프를 갖는다.

**스코프는 "식별자를 검색하는 규칙"이라는 표현이 더 적합하다.**

## 13.4 함수 레벨 스코프

### 블록 레벨 스코프 vs 함수 레벨 스코프

**대부분의 프로그래밍 언어**

- 모든 코드 블록(`if`, `for`, `while`, `try/catch` 등)이 지역 스코프를 만든다
- 이를 **블록 레벨 스코프**라 한다

**JavaScript의 `var` 키워드**

- 함수의 코드 블록만 지역 스코프로 인정한다
- 이를 **함수 레벨 스코프**라 한다

### 예제

```javascript
var x = 1;

if (true) {
  var x = 10;
}

console.log(x); // 10
```

`if` 문 내부의 `x`는 전역 변수다. `var` 키워드는 함수 레벨 스코프만 인정하기 때문에 `if` 문의 코드 블록은 지역 스코프를 생성하지 않는다.

```javascript
for (var i = 0; i < 5; i++) {
  // ...
}

console.log(i); // 5 - i는 전역 변수
```

`for` 문 내부의 `i`도 전역 변수가 된다.

### var vs let/const

| 키워드         | 스코프 레벨      |
| -------------- | ---------------- |
| `var`          | 함수 레벨 스코프 |
| `let`, `const` | 블록 레벨 스코프 |

```javascript
function example() {
  if (true) {
    var varVariable = "var";
    let letVariable = "let";
    const constVariable = "const";
  }

  console.log(varVariable); // 'var' - 접근 가능
  console.log(letVariable); // ReferenceError
  console.log(constVariable); // ReferenceError
}
```

함수 내부에서 선언된 경우:

- `var`: 함수 전체에서 유효 (함수 레벨)
- `let`, `const`: 선언된 블록 내에서만 유효 (블록 레벨)

## 13.5 렉시컬 스코프

```javascript
var x = 1;

function foo() {
  var x = 10;
  bar();
}

function bar() {
  console.log(x);
}

foo(); // 1
bar(); // 1
```

### 두 가지 스코프 결정 방식

1. **동적 스코프 (Dynamic Scope)**
   - 함수를 어디서 **호출**했는지에 따라 상위 스코프 결정
   - `bar()` 함수가 `foo()` 내부에서 호출되었으므로 `foo`의 지역 스코프를 상위 스코프로 참조 → `x`는 10

2. **렉시컬 스코프 (Lexical Scope) = 정적 스코프 (Static Scope)**
   - 함수를 어디서 **정의**했는지에 따라 상위 스코프 결정
   - `bar()` 함수가 전역에서 정의되었으므로 전역 스코프를 상위 스코프로 참조 → `x`는 1

### JavaScript는 렉시컬 스코프를 따른다

**자바스크립트는 렉시컬 스코프를 사용하기 때문에 함수의 상위 스코프는 호출 위치가 아니라 함수가 정의된 위치를 기준으로 결정된다.**

**함수의 상위 스코프는 언제나 자신이 정의된 스코프다.**

### 렉시컬 스코프의 동작 과정

1. **함수 정의**는 코드 평가 단계에서 처리됨
2. 이때 **함수 객체가 생성**됨
3. 동시에 **상위 스코프 참조**가 저장됨
4. 이후 호출 시점과는 무관

> 렉시컬 스코프는 클로저와 깊은 관계가 있다. 24장에서 나옴
