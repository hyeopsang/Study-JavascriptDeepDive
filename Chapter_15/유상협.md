# 15장 let, const 키워드와 블록 레벨 스코프

### 함수 레벨 스코프 (Function-level Scope)

- **대표 키워드:** `var`
- **특징:** 오직 **함수(Function)**의 코드 블록만 지역 스코프로 인정합니다.
- **문제점:** `if` 문, `for` 문, `while` 문 등의 중괄호 `{}` 안에서 선언해도, 함수 안이라면 **블록 밖에서 접근이 가능**합니다. (의도치 않은 변수 값 변경의 원인이 됩니다.)

### 블록 레벨 스코프 (Block-level Scope)

- **대표 키워드:** `let`, `const`
- **특징:** **모든 코드 블록**(`{}` 중괄호로 감싸진 모든 것)을 지역 스코프로 인정합니다.
- **장점:** 함수뿐만 아니라 `if`, `for` 등의 블록 내부에서 선언된 변수는 **블록 밖에서 접근할 수 없습니다.** (더 안전하고 예측 가능합니다.)

## `let` 키워드

### 변수 중복 선언 금지

`var` 키워드와 달리 `let`은 동일한 스코프 내에서 같은 이름의 변수를 **다시 선언할 수 없습니다.** 만약 중복 선언을 시도하면 문법 에러(`SyntaxError`)가 발생하여 코드가 실행되지 않습니다.

### 변수 호이스팅 (Hoisting)

`let` 키워드도 호이스팅이 발생하지만, `var`와 다르게 동작합니다.

- **일시적 사각지대 (Temporal Dead Zone, TDZ):** 선언 단계와 초기화 단계가 분리되어 있습니다. 스코프의 시작 지점부터 변수 선언문(초기화)에 도달하기 전까지는 변수를 참조할 수 없으며, 이 구간을 TDZ라고 부릅니다.
- 이 구간에서 변수에 접근하면 참조 에러(`ReferenceError`)가 발생합니다.

## `const` 키워드

### 선언과 초기화

`const` 키워드로 선언한 변수는 **반드시 선언과 동시에 초기화**해야 합니다. 그렇지 않으면 문법 에러(`SyntaxError`)가 발생합니다.

> 예: const foo = 1; (O), const bar; (X)

### 재할당 금지

`const`로 선언된 변수는 값을 변경할 수 없습니다. 즉, 처음 할당된 값 이후에 새로운 값을 **재할당하는 것이 금지**됩니다. 이를 시도하면 타입 에러(`TypeError`)가 발생합니다.

### 상수 (Constant)

일반적으로 **상수**란 변하지 않는 고정된 값을 의미합니다.

- `const`를 사용하여 원시 값(숫자, 문자열 등)을 할당하면, 원시 값 자체도 변경 불가능하고 재할당도 금지되므로 완벽한 상수로 사용할 수 있습니다.
- 상수의 이름은 **대문자와 스네이크 케이스**(예: `MAX_LEVEL`)를 사용하여 표현합니다.

### `const` 키워드와 객체

`const` 키워드는 **재할당을 금지할 뿐, "불변"을 의미하지는 않습니다.**

- `const` 변수에 **객체**를 할당한 경우, 변수에 저장된 참조 값(주소)은 바꿀 수 없지만 **객체 내부의 프로퍼티는 변경(추가, 삭제, 수정)할 수 있습니다.**
- 즉, 객체 자체를 다른 객체로 교체하는 것은 불가능하지만, 내용물은 바꿀 수 있습니다.
