# 23장 실행 컨텍스트

## 23.1 소스코드의 타입

전역 코드: 전역에 존재하는 소스코드,전역에 정의된 함수,클래스의 내부 코드는 포함x
함수 코드: 함수 내부에 존재하는 소스코드를 말한다. 함수내부에 중첩된 함수,클래스는 포함 x
eval 코드: 빌트인 전역 함수인 eval 함수에 인수로 전달되어 실행되는 소스코드
모듈 코드: 모듈 내부에 존재하는 소스코드,모듈 내부의 함수,클래스는 포함x

소스 코드 타입에 따라 실행 선텍스트를 생성하는 과정과 관리 내용이 다름
전역코드->전역 실행 컨텍스트
함수코드->함수 실행 컨텍스트
eval코드 -> eval 실행 컨텍스트
모듈코드 -> 모듈실행 컨텍스트

## 23.2 소스코드의 평가와 실행

소스코드 평가 과정에서는 실행 컨텍스트를 생성하고 변수, 함수등의 선언문만 먼저 실행하여 생성된 변수나 식별자를 키로 실행 컨텍스트가 관리하는 스코프에 등록한다

평가 과정이 끝나면 런타임이 시작되어 함수가 실행되고 값의 변경과 같은 실행 결과는 다시 실행 컨텍스트가 관리하느 스코프에 등록된다

```javascript
const a = 1;
function foo() {
  console.log(a);
}
foo();
// 1. 평가: a라는 변수 이름과 foo라는 함수 이름을 스코프에 등록합니다.
// 2. 실행: a에 1을 할당합니다. foo()를 호출합니다.
// 3 .다시 평가(foo): foo 함수 내부를 평가합니다.
// 4. 다시 실행(foo): console.log(a)를 실행하기 위해 스코프에서 a를 찾아 출력합니다.
```

## 23.3 실행 컨텍스트의 역할

실행 컨텍스트는 식별자를 등록하고 관리하는 스코프와 코드 실행 순서를 관리를 구현한 내부 매커니즘으로 모든 코드는 실행 컨텍스트에 등록된다

```javascript
const x = 1;
function foo(a) {
  const x = 10;
  console.log(a + x); // 110
}
foo(100);
console.log(x); // 1
// 1. 전역 변수와 함수를 전역 스코프에 등록한다
// 2. 전역 변수는 값이 할당되고 함수는 호출된다 함수가 호출되면 전역 코드의 실행을 중단하고 함수 내부로 들어간다
// 3. 매개변수와 지역 변수 선언문이 매개변수와 지역 변수가 실행 컨텍스트가 관리하는 지역 스코프에 등록
// 4. 함수 코드 실행하여 매개변수와 지역 변수에 값이 할당되고 console.log 메서드가 호출된다.
```

## 23.4 실행 컨텍스트 스택

```javascript
const x = 1;
function foo() {
  const y = 2;
  function bar() {
    const z = 3;
    console.log(x + y + z);
  }
  bar();
}
foo(); // 6
```

1. **전역 코드의 평가와 실행**
   내용: 전역 실행 컨텍스트를 생성하여 **스택에 푸시(Push)**합니다. 전역 스코프에 x와 foo를 등록한 뒤, 런타임에 x에 1을 할당하고 foo()를 호출합니다.

2. **foo 함수 코드의 평가와 실행**
   내용: foo()가 호출되면 전역 코드 실행을 중단하고 foo 실행 컨텍스트를 생성하여 스택에 푸시합니다. 지역 변수 y와 중첩 함수 bar를 foo 스코프에 등록한 뒤, y에 2를 할당하고 bar()를 호출합니다.

3. **bar 함수 코드의 평가와 실행**
   내용: bar 실행 컨텍스트를 생성하여 스택에 푸시합니다. 지역 변수 z를 bar 스코프에 등록하고 z에 3을 할당합니다. console.log(x + y + z)를 실행하기 위해 스코프 체인을 통해 x, y, z 값을 모두 찾아내 출력한 뒤 bar는 종료됩니다.

4. **foo 함수 코드로 복귀**
   내용: bar 함수가 종료되면 스택에서 **제거(Pop)**됩니다. 제어권은 다시 foo 함수로 돌아오며, foo 또한 더 이상 실행할 코드가 없으므로 종료됩니다.

5. **전역 코드로 복귀 및 종료**
   내용: foo 실행 컨텍스트가 스택에서 **제거(Pop)**됩니다. 다시 전역으로 돌아와 실행할 코드가 남아있는지 확인한 뒤, 없으면 전역 실행 컨텍스트도 제거되어 스택은 완전히 비워지게 됩니다.

## 23.5 렉시컬 환경

렉시컬 환경은 식별자와 식별자에 바인딩 된 값, 상위 스코프에 대한 참조를 기록하는 자료구조로 실행 컨텍스트를 구성하는 컴포넌트다. 실행 컨텍스트 스택이 코드의 실행 순서를 관리한다면 렉시컬 환걍은 스코프와 식별자를 관리한다

렉시컬 환경은 키와 값을 갖는 객체 형태의 스코프(전역, 함수, 블록 스코프)를 생성하여 식별자를 키로 등록하고 식별자에 바인딩된 값을 관리한다

렉시컬 환경은 두 개의 컴포넌트로 구성된다

1. 환경 레코드 Environment Record
   스코프에 포함된 식별자를 등록하고 등록된 식별자에 바인딩된 값을 관리하는 저장소다. 환경 레코드는 소스코드의 타입에 따라 관리하는 내용에 차이가 있다

2. 외부 렉시컬 환경에 대한 참조 Outer Lexical Environment Reference
   외부 렉시컬 환경에 대한 참조는 상위 스코프를 가리킨다. 이때 상위 스코프란 외부 렉시컬 환경, 즉 해당 실행 컨텍스트를 생성한 소스코드를 포함하는 상위 코드의 렉시컬 환경을 말한다. 외부 렉시컬 환경에 대한 참조를 통해 단방향 링크드 리스트인 스코프 체인을 구현한다

## 23.6 실행 컨텍스트의 생성과 식별자 검색과정

### 23.6.1 전역 객체 생성

전역 객체는 전역 코드가 평가되기 이전에 생성된다

```javascript
window.toString(); // -> ["object Window"]
window.__proto__.__proto__.__proto__.__proto__.__proto__ === Object.protytpe; // true
```

### 23.6.2 전역 코드 평가
**1. 전역 실행 컨텍스트 생성**

비어있는 전역 실행 컨텍스트를 생성하여 **실행 컨텍스트 스택(Call Stack)**에 푸시합니다.

이 컨텍스트는 실행 중인 실행 컨텍스트(running execution context)가 됩니다.

**2. 전역 렉시컬 환경 생성**

전역 렉시컬 환경(Global Lexical Environment)을 생성하고 전역 실행 컨텍스트에 바인딩합니다.

**2.1 전역 환경 레코드 생성**

전역 변수와 전역 객체를 관리하며, ES6를 기점으로 두 가지 레코드로 구성됩니다.

**2.1.1 객체 환경 레코드 생성**

대상: var 키워드 변수, 함수 선언문, 빌트인 전역 프로퍼티/함수.

특징: BindingObject를 통해 **전역 객체(window)**의 프로퍼티와 메서드가 됩니다.

효과: window.x 처럼 전역 객체를 통해 식별자에 접근할 수 있는 이유입니다.

**2.1.2 선언적 환경 레코드 생성**

대상: let, const 키워드로 선언한 전역 변수(함수 표현식 포함).

특징: 전역 객체에 등록되지 않고 별도의 개념적 블록 내에 저장됩니다.

주의: 선언과 초기화가 분리되어 진행되므로 런타임 이전 참조 시 **TDZ(일시적 사각지대)**에 빠집니다.

**2.2 this 바인딩**

전역 환경 레코드의 [[GlobalThisValue]] 내부 슬롯에 this가 바인딩됩니다.

참고: 객체 환경 레코드나 선언적 환경 레코드 자체에는 this 바인딩이 존재하지 않습니다.

**2.3 외부 렉시컬 환경에 대한 참조 결정**

현재 평가 중인 코드를 포함하는 상위 스코프의 렉시컬 환경을 가리킵니다.

전역 코드는 스코프 체인의 종점이므로 이 값은 null이 할당됩니다.

### 23.6.3 전역 코드 실행

```javascript
var x = 1;
const y = 2;
function foo() {
  var x = 3;
  const y = 4;
  function bar(b) {
    const z = 5;
    console.log(a + b + x + y + z);
  }
  bar(10);
}
foo(20); // 42
```

변수 할당문 또는 함수 호출문을 실행하려면 변수 또는 함수 이름이 선언된 식별자인지 확인해야 한다. 어느 스코프의 식별자를 참조하면 되는지 결정할 필요가 있는데 이를 식별자 결정이라한다. 식별자 결정을 위해 식별자를 검색할 때는 실행 중인 실행 컨텍스트에서 식별자를 검색하기 시작한다. 선언된 식별자는 실행 컨텍스트의 렉시컬 환경의 환경 레코드에 등록되어 있다

### 23.6.4 foo 함수 코드 평가

foo 함수가 호출되면 전역 코드의 실행을 일시 중단하고 foo 함수 내부로 코드의 제어권이 이동하여 함수 코드를 평가한다

1. 함수 실행 컨텍스트 생성
   foo함수 실행 컨텍스트를 생성한다. 생성된 함수 실행 컨텍스트는 함수 렉시컬 환경이 완성된 다음 실행 컨텍스트에 푸시된다.

2. 함수 렉시컬 환경 생성
   foo 함수 렉시컬 환경을 생성하고 foo 함수 실행 컨텍스트에 바인딩한다.

2.1 함수 환경 레코드 생성
함수 렉시컬 환경을 구성하는 컴포넌트 중 하나인 함수 환경 레코드는 매개변수, arguments 객체, 함수 내부에서 선언한 지역 변수와 중첩 함수를 등록하고 관리한다.

2.1 this 바인딩
함수 환경 레코드의 [[ThisValue]] 내부 슬롯에 this가 바인딩된다. foo 함수는 일반 함수로 호출되었으므로 this는 전역 객체를 가리킨다. 따라서 함수 환경 레코드의 [[ThisValue]] 내부 슬롯에는 전역 객체가 바인딩된다.

2.3 외부 렉시컬 환경에 대한 참조 생성
외부 렉시컬 환경에 대한 참조에 foo 함수 정의가 평가된 시점에 실행 중인 실행 컨텍스트의 렉시컬 환경의 참조가 할당된다.
foo 함수는 전역 함수이므로 전역 코드 평가 시점에 평가된다. 이 시점의 실행 컨텍스트는 전역 실행 컨텍스트다. 따라서 외부 렉시컬 환경에 대한 참조에는 전역 렉시컬 환경의 참조가 할당된다.

### 23.6.5 foo 함수 코드 실행

식별자 결정을 위해 실행 중인 실행 컨텍스트의 렉시컬 환경에서 식별자를 검색하기 시작한다.
검색할 수 없다면 외부 렉시컬 환경에 대한 참조가 가리키는 렉시컬 환경으로 이동해 식별자를 검색한다. 검색할 수 있다면 검색된 식별자에 값을 바인딩한다.

### 23.6.6 bar 함수 코드 평가

bar 함수가 호출되면 bar 함수 내부로 코드의 제어권이 이동하여 bar 함수 코드를 평가하기 시작한다.

### 23.6.7 bar 함수 코드 실행

1. console 식별자 검색
   먼저 console 식별자를 스코프 체인에서 검색한다. 식별자를 검색할 때는 언제나 현재 실행 중인 실행 컨텍스트의 렉시컬 환경에서 검색한다.
   실행 중인 실행 컨텍스트는 bar 함수 실행 컨텍스트이므로 bar 함수 렉시컬 환경에서 console 식별자를 검색한다. 만약 없다면 스코프 체인을 따라 상위 스코프로 이동해 검색한다.

console 식별자는 객체 환경 레코드의 BindingObject를 통해 전역 객체에서 찾을 수 있다.

2. log 메서드 검색
   console 식별자에 바인딩된 객체, 즉 console 객체에서 log 메서드를 검색한다. 이때 console 객체의 프로토타입 체인을 통해 메서드를 검색한다. log 메서드는 상속된 프로퍼티가 아니라 console 객체가 직접 소유하는 프로퍼티다.

3. 표현식 a+b+x+y+z의 평가
   표현식 a+b+x+y+z를 평가하기 위해 식별자를 검색한다. 식별자는 스코프 체인, 즉 현재 실행 중인 실행 컨텍스트의 렉시컬 환경에서 시작해 외부 렉시컬 환경에 대한 참조로 이어지는 렉시컬 환경의 연속에서 검색한다.

4. console.log 메서드 호출

### 23.6.8 bar 함수 코드 실행 종료

console.log 메서드가 호출되고 종료하면 더이상 실행할 코드가 없으므로 실행이 종료된다. 실행 컨텍스트 스택에서 bar 함수 실행 컨텍스트가 pop되어 제거되고 foo 실행 컨텍스트가 실행 중인 실행 컨텍스트가 된다.

### 23.6.9 foo 함수 코드 실행 종료

bar 함수가 종료되면 더이상 실행할 코드가 없으므로 foo 함수 코드의 실행이 종료된다.실행 컨텍스트 스택에서 foo 함수 실행 컨텍스트가 pop되어 제거되고 전역 실행 컨테스트가 실행 중인 실행 컨텍스트가 된다.

### 23.6.10 전역 코드 실행 종료

foo 함수가 종료되면 더이상 실행할 코드가 없으므로 전역 코드의 실행이 종료되고 전역 실행 컨텍스트도 스택에서 pop되어 제거된다. 실행 컨텍스트 스택에는 아무것도 남지 않게 된다

## 23.7 실행 컨텍스트와 블록 레벨 스코프

```javascript
let x = 1;

if (true) {
  let x = 10;
  console.log(x); //10
}
console.log(x); //1
```

if 문의 코드 블록이 실행되면 if 문의 코드 블록을 위한 블록 레벨 스코프를 생성해야 한다
선언적 환경 레코드를 갖는 렉시컬 환경을 새롭게 생성해 기존의 전역 렉시컬 환경을 교체한다
