# 25장 클래스

### 1.1 클래스는 일급 객체

클래스는 일급 객체로, 다음과 같은 특징을 가진다.

- 무명의 리터럴로 생성 가능 (런타임에 생성 가능)
- 변수나 자료구조에 저장 가능
- 함수의 매개변수로 전달 가능
- 함수의 반환값으로 사용 가능

### 1.2 클래스 몸체에서 정의 가능한 메서드 종류

클래스 몸체에는 0개 이상의 **메서드**만 정의할 수 있다. 메서드는 다음 세 가지가 있다.

- `constructor`(생성자)
- 프로토타입 메서드
- 정적 메서드

```javascript
class Person {
  // 생성자
  constructor(name) {
    this.name = name;
  }

  // 프로토타입 메서드
  sayHi() {
    console.log(`Hi! My name is ${this.name}`);
  }

  // 정적 메서드
  static sayHello() {
    console.log("Hello!");
  }
}

const me = new Person("Lee");
console.log(me.name); // Lee
me.sayHi(); // Hi! My name is Lee
Person.sayHello(); // Hello!
```

## 2. 클래스 호이스팅

### 2.1 클래스는 함수로 평가된다

클래스는 **함수로 평가**된다. 이는 다음 코드로 확인할 수 있다.

```javascript
class Person {}
console.log(typeof Person); // function
```

클래스 선언문으로 정의한 클래스는 런타임 이전에 평가되어 **함수 객체(= constructor)** 를 생성한다.

### 2.2 클래스와 프로토타입은 “쌍”으로 만들어진다

생성자 함수로 호출할 수 있는 함수, 즉 `constructor`는 **함수 정의가 평가되어 함수 객체를 생성하는 시점에 프로토타입도 더불어 생성**한다.
이는 **프로토타입과 생성자 함수가 항상 쌍으로 존재**하기 때문이다.

### 2.3 클래스 호이스팅의 핵심: TDZ 때문에 “정의 이전 참조 불가”

클래스는 정의 이전에 참조할 수 없다.

```javascript
console.log(Person);
// ReferenceError: Cannot access 'Person' before initialization
class Person {}
```

이는 클래스가 `let`, `const` 키워드로 선언한 변수처럼 호이스팅되기 때문이다.
즉, 클래스 선언문 이전에 **일시적 사각지대(Temporal Dead Zone; TDZ)** 에 빠져 참조 에러가 발생한다.

```javascript
const Person = "";

{
  // 호이스팅이 발생하지 않는다면 ''이 출력되어야 한다.
  console.log(Person);
  // ReferenceError: Cannot access 'Person' before initialization

  class Person {}
}
```

## 3. 클래스의 인스턴스 생성 과정

클래스의 인스턴스 생성 과정은 생성자 함수의 인스턴스 생성 과정과 매우 유사하다.
`new` 연산자와 함께 클래스를 호출하면 클래스의 내부 메서드 `[[Construct]]`가 호출되어 인스턴스가 생성된다.

큰 흐름은 **3단계**로 정리할 수 있다.

### 3.1 인스턴스 생성과 this 바인딩

`new`와 함께 클래스를 호출하면 다음이 진행된다.

1. **암묵적으로 빈 객체**가 생성된다. 이 빈 객체가 (아직 완성되지 않은) 인스턴스다.
2. 이 인스턴스의 프로토타입으로 **클래스의 `prototype` 프로퍼티가 가리키는 객체**가 설정된다.
3. 암묵적으로 생성된 빈 객체(인스턴스)가 **`this`에 바인딩**된다.
   → 따라서 `constructor` 내부의 `this`는 “클래스가 생성 중인 인스턴스”를 가리킨다.

> 포인트: `constructor` 코드가 실행되기도 전에, 이미 인스턴스와 프로토타입 연결(`[[Prototype]]`)이 끝나 있고, 그 인스턴스가 `this`로 들어온다.

### 3.2 인스턴스 초기화

이제 `constructor` 내부 코드가 실행되며, `this`에 바인딩된 인스턴스를 초기화한다.

- `this`에 프로퍼티를 추가한다.
- 전달받은 인수로 프로퍼티 값을 초기화한다.
- 클래스에 `constructor`가 없다면 이 과정은 생략된다.

### 3.3 인스턴스 반환

모든 처리가 끝나면 **완성된 인스턴스가 바인딩된 `this`가 암묵적으로 반환**된다.

```javascript
class Person {
  constructor(name) {
    // 1. 암묵적으로 인스턴스가 생성되고 this에 바인딩된다.
    console.log(this); // Person {}
    console.log(Object.getPrototypeOf(this) === Person.prototype); // true

    // 2. this에 바인딩되어 있는 인스턴스를 초기화한다.
    this.name = name;

    // 3. 완성된 인스턴스가 바인딩된 this가 암묵적으로 반환된다.
  }
}

const me = new Person("Lee");
console.log(me); // Person { name: "Lee" }
```

#### 3.3.1 constructor에서 “명시적 반환”을 하면 어떻게 되나?

- **객체를 반환하면**: 암묵적 `this` 반환이 무시되고, **그 객체가 반환**된다.

```javascript
class Person {
  constructor(name) {
    this.name = name;
    return {}; // 명시적으로 빈 객체를 반환
  }
}

const me = new Person("Lee");
console.log(me); // {}
```

- **원시값을 반환하면**: 반환이 무시되고 **암묵적으로 `this`가 반환**된다.

```javascript
class Person {
  constructor(name) {
    this.name = name;
    return 100; // 원시값을 반환
  }
}

const me = new Person("Lee");
console.log(me); // Person { name: "Lee" }
```

## 4. 프로퍼티

클래스에서는 다양한 유형의 프로퍼티를 정의할 수 있다.
여기서는 **인스턴스 프로퍼티, 접근자 프로퍼티, 클래스 필드, private 필드, static 필드**를 다룬다.

### 4.1 인스턴스 프로퍼티

```javascript
class Person {
  constructor(name) {
    // 인스턴스 프로퍼티
    this.name = name;
  }
}

const me = new Person("Lee");
console.log(me); // Person {name: "Lee"}
```

- 인스턴스 프로퍼티는 `constructor` 내부에서 정의한다.
- `constructor` 내부의 `this`는 인스턴스를 가리킨다.

### 4.2 접근자 프로퍼티 (getter / setter)

- **Getter (get age()):** 객체의 속성값(user1.age)을 읽으려고 할 때 자동으로 호출되는 함수입니다.

- **Setter (set age(value)):** 객체의 속성에 값(user1.age = -1)을 할당하려고 할 때 자동으로 호출되는 함수입니다. 주로 값의 유효성을 검사(예: 나이가 음수인지 확인)할 때 사용합니다

```javascript
class User {
  constructor(firstName, lastName, age) {
    this.firstName = firstName;
    this.lastName = lastName;
    this.age = age;
  }

  get age() {
    return this.age;
  }

  set age(value) {
    this.age = value;
  }
}

const user1 = new User("Steve", "Job", -1);
console.log(user1.age);
```

- set age(value) 함수 내부를 보면 this.age = value;라고 되어 있습니다.
- 여기서 this.age = ...를 수행하는 순간, 다시 Setter인 set age()가 호출됩니다.
- Setter -> Setter 호출 -> Setter 호출... 이 반복되면서 결국 "Maximum call stack size exceeded" 에러가 발생하며 프로그램이 멈춥니다. Getter도 마찬가지로 return this.age;를 하는 순간 다시 Getter를 호출하게 됩니다.

위의 문제를 마주치지 않으려고 생긴 명명 규칙

```javascript
class User {
  constructor(firstName, lastName, age) {
    this.firstName = firstName;
    this.lastName = lastName;
    this.age = age; // 여기서 setter가 호출됨
  }

  get age() {
    return this._age; // _age라는 별도의 공간에서 값을 가져옴
  }

  set age(value) {
    // 유효성 검사 가능
    this._age = value < 0 ? 0 : value; // _age라는 별도의 공간에 값을 저장
  }
}
```

### 4.3 클래스 필드 정의

```javascript
class Person {
  // 클래스 필드 정의
  name = "Lee";

  constructor() {
    console.log(name); // ReferenceError: name is not defined
  }
}

const me = new Person();
console.log(me); // Person {name: "Lee"}
```

- 클래스 필드는 클래스가 생성할 인스턴스의 프로퍼티를 가리키는 용어다.
- 클래스 필드를 정의할 때는 **this를 사용하지 않는다.**
- `this`는 `constructor`와 메서드 내에서만 유효하다.

#### 클래스 필드에 함수를 할당하는 방식(인스턴스 메서드가 됨)

```javascript
class Person {
  // 클래스 필드에 함수를 할당
  getName = function () {
    return this.name;
  };
  // 화살표 함수로 정의할 수도 있다.
  // getName = () => this.name;
}
```

이 경우 인스턴스 메서드가 되므로 권장하지 않는다.

### 4.4 private 필드 정의 제안 (#)

```javascript
class Person {
  // private 필드 정의
  #name = ""; // 1. 반드시 클래스 최상단(필드 영역)에 먼저 선언해야 함

  constructor(name) {
    // private 필드 참조
    this.#name = name; // 2. 그 다음 constructor 내부에서 할당 및 참조가 가능함
  }
}

const me = new Person("Lee");
console.log(me.#name);
// SyntaxError: Private field '#name' must be declared in an enclosing class
```

### 4.5 static 필드 정의 제안

```javascript
class MyMath {
  // static public 필드 정의
  static PI = 22 / 7;

  // static private 필드 정의
  static #num = 10;

  // static 메서드
  static increment() {
    return ++MyMath.#num;
  }
}

console.log(MyMath.PI); // 3.142857142857143
console.log(MyMath.increment()); // 11
```

- 정적 필드는 인스턴스가 아닌 **클래스 이름으로 접근**한다.

## 5. 상속에 의한 클래스 확장

상속에 의한 클래스 확장은 기존 클래스를 상속받아 새로운 클래스를 확장하여 정의하는 것이다.
이는 프로토타입 기반 상속과는 다른 개념이다.

### 5.1 클래스 상속 예시

```javascript
class Animal {
  constructor(age, weight) {
    this.age = age;
    this.weight = weight;
  }

  eat() {
    return "eat";
  }
  move() {
    return "move";
  }
}
// 클래스 확장
class Bird extends Animal {
  fly() {
    return "fly";
  }
}

const bird = new Bird(1, 5);
console.log(bird); // Bird {age: 1, weight: 5}
console.log(bird instanceof Bird); // true
console.log(bird instanceof Animal); // true

console.log(bird.eat()); // eat
console.log(bird.move()); // move
console.log(bird.fly()); // fly
```

클래스 확장은 코드 재사용 관점에서 매우 유용하다.

### 5.2 extends 키워드와 용어

```javascript
class Base {}
class Derived extends Base {}
```

- 상속으로 확장된 클래스: **서브클래스(파생/자식)**
- 상속해주는 클래스: **수퍼클래스(베이스/부모)**

### 5.3 동적 상속 (extends 뒤에 “표현식” 가능)

`extends`는 클래스뿐만 아니라 생성자 함수를 상속받아 확장할 수도 있다.

```javascript
function Base(a) {
  this.a = a;
}

class Derived extends Base {}

const derived = new Derived(1);
console.log(derived); // Derived {a: 1}
```

또한 `extends` 뒤에는 클래스뿐만 아니라 `[[Construct]]` 내부 메서드를 갖는 함수 객체로 평가될 수 있는 **모든 표현식**을 사용할 수 있다.

```javascript
function Base1() {}
class Base2 {}

let condition = true;

class Derived extends (condition ? Base1 : Base2) {}

const derived = new Derived();
console.log(derived); // Derived {}
console.log(derived instanceof Base1); // true
console.log(derived instanceof Base2); // false
```

### 5.4 서브클래스의 constructor 기본 형태

서브클래스에서 `constructor`를 생략하면 다음과 같은 형태가 암묵적으로 정의된다.

```javascript
constructor(...args) { super(...args); }
```

- ...args (Rest Parameter): 서브클래스를 생성할 때 전달된 모든 인자를 배열로 묶어서 받겠다는 뜻이다.
- super(...args): 부모 클래스(Superclass)의 생성자를 호출하면서, 방금 받은 모든 인자를 그대로 부모에게 전달하겠다는 뜻이다. (배열을 다시 개별 인자로 펼쳐서 전달한다.)

### 5.5 super 키워드

`super`는 함수처럼 호출할 수도 있고 `this`처럼 식별자처럼 참조할 수 있는 특수한 키워드다.

#### 5.5.1 super 호출 (수퍼클래스 constructor 호출)

```javascript
class Base {
  constructor(a, b) {
    this.a = a;
    this.b = b;
  }
}

class Derived extends Base {
  constructor(a, b, c) {
    super(a, b);
    this.c = c;
  }
}

const derived = new Derived(1, 2, 3);
console.log(derived); // Derived {a: 1, b: 2, c: 3}
```

규칙/주의사항:

- 서브클래스에서 `constructor`를 생략하지 않는 경우, 서브클래스 `constructor`에서는 반드시 `super()`를 호출해야 한다.
- `super()` 호출 전에는 `this`를 참조할 수 없다.
- `super`는 반드시 서브클래스 `constructor`에서만 호출한다.

#### 5.5.2 super 참조 (수퍼클래스 메서드 호출)

```javascript
class Base {
  sayHi() {
    return "Hi!";
  }
}

class Derived extends Base {
  sayHi() {
    return `${super.sayHi()} How are you doing?`;
  }
}

const derived = new Derived();
console.log(derived.sayHi()); // Hi! How are you doing?
```

메서드 내에서 `super`를 참조하면 수퍼클래스의 메서드를 호출할 수 있다.
만약 `super`를 참조하지 않으면 오버라이딩이 일어난다.

### 5.6 상속 클래스의 인스턴스 생성 과정

서브클래스의 인스턴스 생성은 “수퍼클래스 생성 → 서브클래스 확장 초기화” 순서로 진행된다.

- 서브클래스의 `super` 호출
- 수퍼클래스의 인스턴스 생성과 this 바인딩
- 수퍼클래스의 인스턴스 초기화
- 서브클래스 constructor로의 복귀와 this 바인딩
- 서브클래스의 인스턴스 초기화
- 인스턴스 반환

> 핵심 감각: 서브클래스는 “내가 this 만들게요”가 아니라, **super가 만든 this를 받아서 확장**하는 흐름이다. 그래서 `super()` 전에는 `this`를 못 쓴다.

## 6. 표준 빌트인 생성자 함수 확장

```javascript
class MyArray extends Array {
  uniq() {
    return this.filter((v, i, self) => self.indexOf(v) === i);
  }

  average() {
    return this.reduce((pre, cur) => pre + cur, 0) / this.length;
  }
}

const myArray = new MyArray(1, 1, 2, 3);
console.log(myArray.uniq()); // MyArray(3) [1, 2, 3]
console.log(myArray.average()); // 1.75
```

`String`, `Number`, `Array` 같은 표준 빌트인 객체도 `[[Construct]]` 내부 메서드를 갖는 생성자 함수이므로 `extends` 키워드를 사용하여 확장할 수 있다.
