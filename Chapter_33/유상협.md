# 33장 Symbol

심벌(Symbol)은 원시 값이며 다른 값과 중복 되지 않는 유일무이한 값이다.

```javascript
const mySymbol1 = Symbol("mySymbol");
const mySymbol2 = Symbol("mySymbol");

console.log(mySymbol1 === mySymbol2); // false
```

위에서 설명 했던 다른 값과 중복 되지 않는 다는 말에 대한 예제이다.
"mySymbol"이라는 똑같은 문자로 값을 생성했지만, mySymbol1과 mySymbol2는 서로 다른 값으로 판정이 난다.

심벌 값도 문자열, 숫자, 불리언과 같이 객체처럼 접근하면 암묵적으로 래퍼 객체를 생성한다.
decription 프로퍼티와 toString 메서드는 Symbol.prototype의 프로퍼티다.

```javascript
const mySymbol = Symbol("mySymbol");

// 심벌도 래퍼 객체를 생성한다.
console.log(mySymbol.desciption); // mySymbol
console.log(mySymbol.toString()); // Symbol(mySymbol)
```

심벌 값은 암묵적으로 문자열이나 숫자 타입으로 변환되지 않지만, 불리언 타입으로는 암묵적으로 타입 변환된다.

```javascript
const mySymbol = Symbol();

// 심벌 값은 암묵적으로 문자열이나 숫자 타입으로 변환되지 않는다.
console.log(mySymbol + ""); // TypeError : Cannot convert a Symbol calue to a string
console.log(+mySymbol); // TypeError : Cannot convert a Symbol calue to a number

// 불리언 타입으로는 암묵적으로 타입 변환된다.
console.log(!!mySymbol); // true
```

## Symbol.for / Symbol.keyFor 메서드

Symbol.for 메서드는 인수로 전달받은 문자열을 키로 사용하여 키와 심벌 값의 쌍들이 저장되어 있는 전역 심벌 레지스트리에서 해당 키와 일치하는 심벌 값을 검색한다.

- 검색에 성공하면 새로운 심벌 값을 생성하지 않고 검색된 심벌 값을 반환한다.
- 검색에 실패하면 새로운 심벌 값을 생성하여 Symbol.for 메서드의 인수로 전달된 키로 전역 심벌 레지스트리에 저장한 후, 생성한 심벌 값을 반환한다.

```javascript
// 전역 심벌 레지스트리에 mySymbol이라는 키로 저장된 심벌 값이 없기에 새로운 심벌 값을 생성
const s1 = Symbol.for("mySymbol");
// 전역 심벌 세지스트리에 mySymbol이라는 키로 저장된 심벌 값이 있어서 해당 심벌 값을 반환
const s2 = Symbol.for("mySymbol");

console.log(s1 === s2); // true
```

Symbol.for 메서드를 사용하면 애플리케이션 전역에서 중복되지 않는 유일무이한 상수인 심벌 값을 단 하나만 생성하여 전역 심벌 레지스트리를 통해 공유할 수 있다.

Symbol.keyFor 메서드를 사용하면 전역 심벌 레지스트리에 저장된 심벌 값의 키를 추출할 수 있다.

```javascript
// 전역 심벌 레지스트리에 mySymbol 이라는 키로 저장된 심벌 값이 없으면 새로운 심벌 값을 생성
const s1 = Symbol.for("mySymbol");
Symbol.keyFor(s1); // mySymbol

// Symbol 함수를 호출하여 생성한 심벌 값은 전역 심벌 레지스트리에 등록되어 관리되지 않는다.
const s2 = Symbol("foo");
// 전역 심벌 레지스트리에 저장된 심벌 값의 키를 추출
Symbol.keyFor(s2); // undefined -> 따로 관리되지 않는다.
```

## 심벌과 상수

#### 핵심 개념

문제점: 숫자(1)나 문자열('UP')로 상수를 정의하면, 우연히 값이 중복될 경우 서로 다른 상수를 같다고 착각하는 논리적 오류가 발생한다.

#### 해결책

**Symbol**은 생성될 때마다 유일무이한 값을 만든다. 이를 사용하면 값의 중복을 원천적으로 차단할 수 있다.

#### 활용

변경 불가능한 상수 객체(enum 대용)를 만들 때 **Object.freeze**와 함께 사용한다.

#### 문제

값(1)이 겹쳐서 서로 다른 개념(방향과 성별)이 같다고 판단되는 오류가 발생한다.

```javascript
// 숫자 1이 중복되어 사용됨
const Direction = { UP: 1, DOWN: 2 };
const Gender = { MALE: 1, FEMALE: 2 };

// 논리적 오류: "위쪽"과 "남성"은 다르지만, 값(1)이 같아 true가 나온다.
console.log(Direction.UP === Gender.MALE); // true (버그 발생 위험)
```

#### 해결

Symbol()은 호출할 때마다 매번 새로운 값을 생성하므로 절대 겹치지 않는다.

```javascript
// Symbol을 사용하여 유일무이한 값을 생성
const Direction = Object.freeze({
  UP: Symbol("up"),
  DOWN: Symbol("down"),
});

const Gender = Object.freeze({
  MALE: Symbol("male"), // 설명('male')이 같아도
  FEMALE: Symbol("female"), // 실제 값은 완전히 다르다.
});

// "위쪽"과 "남성"은 확실히 다른 값으로 인식된다.
console.log(Direction.UP === Gender.MALE); // false (안전함)
```

## 심벌을 프로퍼티 키로 사용하기

#### 핵심 개념

객체의 프로퍼티 키는 보통 문자열(예: 'name', 'age')을 사용한다. 하지만 심벌 값도 키로 사용할 수 있다.

**충돌 방지:** 심벌은 유일무이하므로, 이미 존재하는 프로퍼티 이름과 절대 겹치지 않는다. 여러 라이브러리가 하나의 객체를 공유할 때 안전하게 자신만의 데이터를 저장할 수 있다.

```javascript
const mySymbol = Symbol("myKey");
const obj = {};

// 1. 심벌을 키로 프로퍼티 정의 (대괄호 사용 필수)
obj[mySymbol] = 123;

// 2. 접근할 때도 대괄호와 심벌 변수가 필요하다.
console.log(obj[mySymbol]); // 123
console.log(obj.mySymbol); // undefined (점 표기법은 문자열 키 'mySymbol'을 찾으므로 안 됨)
```

**은닉성:** 심벌 키로 만든 프로퍼티는 for...in 문이나 Object.keys() 같은 일반적인 조회 방법으로 나타나지 않는다.

심벌 프로퍼티는 겉으로 드러나지 않아, 내부 동작을 위한 값을 저장할 때 좋다.

```javascript
const id = Symbol("id");

const user = {
  name: "Lee",
  age: 30,
  [id]: "my_secret_id_123", // 심벌 키 프로퍼티
};

// 일반적인 반복문에서는 심벌 프로퍼티가 보이지 않는다.
for (const key in user) {
  console.log(key); // name, age 만 출력됨 ('id'는 숨겨짐)
}

// 굳이 찾으려면 전용 메서드를 써야 한다.
console.log(Object.getOwnPropertySymbols(user)); // [Symbol(id)]
```

## Well-knwon Symbol

#### 개념

자바스크립트 엔진의 내부 알고리즘 식별자
자바스크립트는 기본적으로 제공하는 빌트인 심벌 값들이 있다. 이를 Well-known Symbol이라 부른다.

이 심벌들은 자바스크립트 엔진의 내부 알고리즘에 사용되는 일종의 **'예약된 식별자'**다. 예를 들어, Array, String, Map, Set 등 for...of 문으로 순회 가능한 모든 빌트인 이터러블은 **Symbol.iterator**를 키로 갖는 메서드를 가지고 있다.

엔진은 for...of 문이 실행될 때, 해당 객체에 Symbol.iterator라는 키를 가진 메서드가 있는지 확인하고, 있다면 이를 호출하여 순회를 시작한다.

#### 하위 호환성과 확장성

만약 자바스크립트가 순회 기능을 추가하면서 심벌이 아닌 문자열 키(예: "iterator")를 사용했다면, 기존에 "iterator"라는 이름의 프로퍼티를 사용하던 코드와 충돌하여 오류가 발생했을 것이다.

하지만 Well-known Symbol은 유일무이한 심벌 값을 키로 사용하므로, 기존 프로퍼티 키는 물론 미래에 추가될 어떤 프로퍼티 키와도 절대 중복되지 않는다. 이를 통해 기존 코드에 영향을 주지 않으면서(하위 호환성 보장) 새로운 기능을 안전하게 추가할 수 있다.

#### 사용자 정의 이터러블 구현

일반 객체는 이터러블이 아니지만, Well-known Symbol인 Symbol.iterator를 키로 사용하여 메서드를 구현하면 이터러블처럼 동작하게 만들 수 있다.

```javascript
// 1. 이터레이션 프로토콜을 준수하는 객체 구현
const iterable = {
  // Symbol.iterator 메서드를 구현한다.
  // 이 메서드는 이터레이터를 반환해야 한다.
  [Symbol.iterator]() {
    let cur = 1;
    const max = 3;
    // 이터레이터 반환
    return {
      next() {
        return { value: cur++, done: cur > max + 1 };
      },
    };
  },
};

// 2. 이터러블이 되었으므로 for...of 문으로 순회 가능하다.
for (const num of iterable) {
  console.log(num); // 1 2 3
}

// 3. 스프레드 문법 등도 사용 가능하다.
console.log([...iterable]); // [1, 2, 3]
```
