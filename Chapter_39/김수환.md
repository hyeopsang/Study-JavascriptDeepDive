# 39장 DOM

**DOM: HTML 문서의 계층적 구조와 정보를 표현하며 이를 제어할 수 있는 프로퍼티와 메서드를 제공하는 트리 자료구조**

## 39.1 노드

### 39.1.1 HTML 요소와 노드 객체

HTML 요소는 태그, 속성 컨텐츠가 합쳐진 덩어리 ex) <button class='btn'>버튼</button>
태그와 속성은 요소 노드로 컨텐츠는 텍스트 노드로 분리된다. 이때 부자 관계를 형성하여 트리 자료구조가 된다.
노드 객체들로 구성된 트리 자료구조를 DOM이라 한다. 트리구조이기 때문에 DOM트리라고도 한다.

### 39.1.2 노드 객체의 타입

DOM 노드는 12개의 종류가 있지만 중요한 건 4가지이다.

1. 문서 노드: 최상위에 존재하는 루트 노드로 document 객체를 가리킨다.
2. 요소 노드: HTML 요소를 가리키는 객체로 문서의 구조를 표현한다.
3. 어트리뷰트 노드: HTML 요소의 어트리뷰트를 가리키는 객체로 부모 노드와 연결되어 있지 않고 요소 노드에만 연결된다.
4. 텍스트 노드: HTML 요소의 텍스트를 가리키는 객체로 요소 노드의 자식 노드이며 자식이 없는 DOM 트리의 최종단이다.

### 39.1.3 노드 객체의 상속 구조

공통 기능: 모든 노드가 쓰는 기능은 Node에, 모든 요소가 쓰는 기능은 Element나 HTMLElement에 모여 있다.

특수 기능: 특정 태그만 쓰는 기능은 HTML특정 Element에 정의되어 있다.

1. Object: 모든 자바스크립트 객체의 최상위 조상이다.

2. EventTarget: 이벤트를 발생시키는 객체이다.

3. Node: DOM 트리를 구성하는 모든 노드 객체가 공통으로 상속받는 인터페이스이다.

4. Element: 웹 문서의 **'요소(태그)'**를 위한 공통 인터페이스이다. (HTML뿐만 아니라 XML, SVG 요소 포함)

5. HTMLElement : **'HTML 요소'**만이 가지는 공통적인 속성을 정의한다.

6. HTML 특정 Element (예: HTMLInputElement): 각 태그마다 필요한 고유한 기능을 정의하는 최하위 인터페이스이다.

## 39.2 요소 노드 취득

HTML 구조, 내용, 스타일 등을 동적으로 조작하려면 먼저 요소 노드를 취득해야 한다.

### 39.2.1 id를 이용한 노드 취득

Document.prototype.getElementById 메서드를 사용해 id 값을 갖는 하나의 요소 노드를 탐색하여 반환할 수 있다.

**주의:**

1. id 값은 유일해야 한다.
2. 중복된 id를 갖는 요소가 여러 개 존재할 때 첫 번째 요소 노드만 반환한다.
3. id가 존재하지 않으면 null을 반환한다.
4. id 값과 동일한 이름의 전역 변수가 암묵적으로 선언되고, 해당 노드 객체가 할당된다.

### 39.2.2 태그 이름을 이용한 요소 노드 취득

Document.prototype/Element.prototype.getElementsByTagName 메서드를 사용해 인수로 전달한 태그 이름을 갖는 모든 요소 노드를 탐색하여 반환할 수 있다. (HTMLCollection 객체를 반환)

1. 함수는 하나의 값만 반환하므로 여러 개의 값을 반환하려면 배열이나 객체 같은 자료구조에 담아 반환해야 한다.
2. HTMLCollection 객체는 유사 배열 객체이면서 이터러블이다.
3. 모든 요소 노드를 취득하려면 인수로 '\*'을 전달한다.
4. 태그 이름을 갖는 요소가 존재하지 않는 경우 빈 HTMLCollection 객체를 반환한다.

### 39.2.3 class를 이용한 요소 노드 취득

Document.prototype/Element.prototype.getElementsByClassName 메서드를 사용해 인수로 전달한 class 어트리뷰트 값을 갖는 모든 요소 노드들을 탐색하여 반환한다. (HTMLCollection 객체를 반환)

1. 공백으로 구분하여 여러 개의 class를 지정할 수 있다.
2. class 값을 갖는 요소가 존재하지 않는 경우 빈 HTMLCollection 객체를 반환한다.

### 39.2.4 CSS 선택자를 이용한 요소 노드 취득

Document.prototype/Element.prototype.querySelector 메서드를 사용해 인수로 전달한 CSS 선택자를 만족시키는 하나의 요소 노드를 탐색해 반환한다.

1. CSS 선택자를 만족시키는 하나의 요소 노드를 탐색하여 반환한다.
2. 여러 개일 경우, 첫 번째 요소 노드만 반환한다.
3. 존재하지 않는 경우, null을 반환한다.
4. 문법에 맞지 않는 경우 DOMException 에러가 발생한다.

Document.prototype/Element.prototype.querySelectorAll 메서드를 사용해 인수로 전달한 CSS 선택자를 만족시키는 모든 요소 노드를 탐색해 반환한다. querySelectorAll 메서드는 NodeList 객체를 반환한다. (유사 배열 객체이면서 이터러블이다)

1. 요소가 존재하지 않는 경우 빈 NodeList 객체를 반환한다.
2. 문법에 맞지 않는 경우 DOMException 에러가 발생한다.
3. 모든 요소 노드를 취득하려면 인수로 '\*'을 전달한다.

CSS 선택자는 id나 태그이름 메서드보다 느리지만 일관된 방식과 구체적인 조건으로 요소 노드를 취득할 수 있는 장점이 있다.

### 39.2.5 특정 요소 노드를 취득할 수 있는지 확인

Element.prototype.matches 메서드는 인수로 전달한 CSS 선택자를 통해 특정 요소를 취득할 수 있는지 확인한다.

### 39.2.6 HTMLCollection과 NodeList

HTMLCollection과 NodeList는 DOM API가 여러 개의 결과값을 반환하기 위한 DOM 컬렉션 객체로, 유사 배열 객체이면서 이터러블인 살아있는 객체이다.

HTMLCollection

1. getElementsByTagName, getElementsByClassName 메서드가 반환한다.
2. 노드 객체의 상태 변화를 실시간으로 반영하는 살아 있는 DOM 컬렉션 객체이다.
3. for 문으로 순회하면서 노드 객체를 변경해야 할 때 주의해야 한다. 살아있는 객체이기 때문에 실시간으로 인덱스가 재배치된다.

NodeList

1. HTMLCollection 객체의 부작용을 해결하기 위해 querySelectorAll 메서드를 사용하는 방법이 있다. 이 때 NodeList 객체는 non-live 객체이다.
2. childNodes 프로퍼티가 반환하는 NodeList 객체는 HTMLCollection 객체와 같이 live 객체이므로 주의해야 한다.

노드 객체의 상태 변경과 상관없이 안전하게 DOM 컬렉션을 사용하려면 배열로 변환하여 사용하는 것을 권장한다.

## 39.3 노드 탐색

Node, Element 인터페이스는 트리 탐색 프로퍼티를 제공한다.

1. Node.prototype: parentNode, previousSibling, firstChild, childNodes 프로퍼티를 제공한다.
2. Element.prototype: previousElementSibling, nextElementSibling, children 프로퍼티를 제공한다.

모두 접근자 프로퍼티이지만 setter 없이 getter만 존재하여 읽기 전용이다.

### 39.3.1 공백 텍스트 노드

공백 문자는 공백 텍스트 노드를 생성한다.

### 39.3.2 자식 노드 탐색

| 프로퍼티                              | 설명                               | 반환 타입        | 텍스트 노드 포함 |
| ------------------------------------- | ---------------------------------- | ---------------- | ---------------- |
| `Node.prototype.childNodes`           | 자식 노드를 모두 탐색하여 반환한다 | NodeList         | O                |
| `Element.prototype.children`          | 자식 노드를 모두 탐색하여 반환한다 | HTMLCollection   | X                |
| `Node.prototype.firstChild`           | 첫 번째 자식 노드를 반환한다       | 텍스트/요소 노드 | O                |
| `Node.prototype.lastChild`            | 마지막 자식 노드를 반환한다        | 텍스트/요소 노드 | O                |
| `Element.prototype.firstElementChild` | 첫 번째 자식 요소 노드를 반환한다  | 요소 노드        | X                |
| `Element.prototype.lastElementChild`  | 마지막 자식 요소 노드를 반환한다   | 요소 노드        | X                |

### 39.3.3 자식 노드 존재 확인

Node.prototype.hasChildNodes 메서드를 사용하면 자식 노드가 존재하는지 확인할 수 있다.
존재하면 true, 없으면 false를 반환한다. 텍스트 노드를 포함하여 확인하기 때문에 텍스트 노드가 아닌 요소 노드가 존재하는지 확인하려면 children.length 또는 Element 인터페이스의 childElementCount 프로퍼티를 사용한다.

### 39.3.4 요소 노드의 텍스트 노드 탐색

텍스트 노드는 요소 노드의 자식 노드이기 때문에 firstChild 프로퍼티로 접근할 수 있다. 첫 번째 자식 노드를 반환한다.

### 39.3.5 부모 노드 탐색

Node.prototype.parentNode 프로퍼티를 사용하여 부모 노드를 탐색할 수 있다.

### 39.3.6 형제 노드 탐색

어트리뷰트 노드는 반환되지 않는다.

| 프로퍼티                                   | 설명                                    | 텍스트 노드 포함 |
| ------------------------------------------ | --------------------------------------- | ---------------- |
| `Node.prototype.previousSibling`           | 이전 형제 노드를 탐색하여 반환한다      | O                |
| `Node.prototype.nextSibling`               | 다음 형제 노드를 탐색하여 반환한다      | O                |
| `Element.prototype.previousElementSibling` | 이전 형제 요소 노드를 탐색하여 반환한다 | X                |
| `Element.prototype.nextElementSibling`     | 다음 형제 요소 노드를 탐색하여 반환한다 | X                |

## 39.4 노드 정보 취득

`Node.prototype.nodeType`: 노드 타입을 나타내는 상수를 반환한다.
- 요소 노드 타입: 1
- 텍스트 노드 타입: 3
- 문서 노드 타입: 9

`Node.prototype.nodeName`: 노드의 이름을 문자열로 반환한다.
- 요소 노드: 대문자 문자열로 태그 이름("UL", "LI" 등)을 반환한다.
- 텍스트 노드: 문자열 "#text"를 반환한다.
- 문서 노드: 문자열 "#document"를 반환한다.

## 39.5 요소 노드의 텍스트 조작

### 39.5.1 nodeValue

`Node.prototype.nodeValue` 프로퍼티는 setter와 getter 모두 존재하여 참조와 할당 모두 가능하다.

1. 노드 객체의 nodeValue 프로퍼티를 참조하면 텍스트를 반환한다.
2. 문서 노드나 요소 노드의 nodeValue 프로퍼티를 참조하면 null을 반환한다.

텍스트 노드의 값만 변경할 수 있기 때문에 다음 순서로 처리한다.

1. 텍스트를 변경할 요소 노드를 취득하여 텍스트 노드를 탐색한다. 텍스트 노드는 요소 노드의 자식 노드이므로 firstChild 프로퍼티를 사용한다.
2. 탐색한 텍스트 노드의 nodeValue 프로퍼티를 사용하여 텍스트 노드의 값을 변경한다.

### 39.5.2 textContent

`Node.prototype.textContent` 프로퍼티는 setter와 getter 모두 존재하는 접근자 프로퍼티로서 요소 노드의 텍스트와 모든 자손 노드의 텍스트를 취득하거나 변경할 수 있다.

1. 요소 노드의 textContent 프로퍼티를 사용하면 시작 태그와 종료 태그 사이의 텍스트를 모두 반환하지만 HTML 마크업은 무시된다.
2. nodeValue보다 textContent가 더 간단하다.
3. HTML 마크업이 문자열 그대로 인식되어 텍스트로 취급된다.

## 39.6 DOM 조작

### 39.6.1 innerHTML

`Element.prototype.innerHTML` 프로퍼티는 setter와 getter 모두 존재하는 접근자 프로퍼티로서 요소 노드의 HTML 마크업을 취득하거나 변경한다.

1. 요소 노드의 시작 태그와 종료 태그 사이에 포함된 모든 HTML 마크업을 문자열로 반환한다.
2. innerHTML 프로퍼티에 문자열을 할당하면 요소 노드의 모든 자식 노드가 제거되고 할당한 문자열에 포함된 HTML 마크업이 파싱되어 요소 노드의 자식 노드로 DOM에 반영된다.
3. 크로스 사이트 스크립팅 공격에 취약하다. 새로운 요소를 삽입할 때 위치를 지정할 수 없다는 단점도 있다.

### 39.6.2 insertAdjacentHTML 메서드

`Element.prototype.insertAdjacentHTML` 메서드는 기존 요소를 제거하지 않고 위치를 지정해 새로운 요소를 삽입할 수 있다.

1. 두 번째 인수로 전달한 HTML 마크업 문자열을 파싱하고, 그 결과로 생성된 노드를 첫 번째 인수로 전달한 위치에 삽입하여 DOM에 반영한다.
2. innerHTML 프로퍼티보다 효율적이고 빠르다.
3. HTML 마크업 문자열을 파싱하므로 크로스 사이트 스크립팅 공격에 취약하다.

### 39.6.3 노드 생성과 추가

1. 요소 노드 생성: `Document.prototype.createElement(tagname)` 메서드를 사용하여 요소 노드를 생성한다.
2. 텍스트 노드 생성: `Document.prototype.createTextNode(text)` 메서드를 사용하여 텍스트 노드를 생성하여 반환한다.
3. 텍스트 노드를 요소 노드의 자식 노드로 추가: `Node.prototype.appendChild(childNode)` 메서드를 사용하여 호출한 노드의 마지막 자식 노드로 추가한다.
4. 요소 노드를 DOM에 추가: `Node.prototype.appendChild(요소 노드)` 메서드를 사용하여 텍스트 노드와 부자 관계로 연결한 요소 노드를 마지막 자식 요소로 추가한다. (리플로우와 리페인트가 실행된다)

### 39.6.4 복수의 노드 생성과 추가

n번 노드를 생성하여 DOM에 추가하면 리플로우와 리페인트가 n번 일어나서 비효율적이다. `DocumentFragment` 노드는 기존 DOM과 별도로 존재하므로 이를 활용한다.

복수의 노드 생성 및 추가 방법:
1. `Document.prototype.createDocumentFragment` 메서드를 사용해 비어있는 DocumentFragment 노드를 생성한다.
2. DocumentFragment에 자식 노드를 추가한다.
3. DocumentFragment 노드를 기존 DOM에 추가한다.

### 39.6.5 노드 삽입

**마지막 노드로 추가**
- `Node.prototype.appendChild(node)`: 인수로 전달받은 노드를 호출한 노드의 마지막 자식 노드로 DOM에 추가한다.

**지정한 위치에 노드 삽입**
- `Node.prototype.insertBefore(newNode, childNode)`: 첫 번째 인수로 전달받은 노드를 두 번째 인수로 전달받은 노드 앞에 삽입한다.
- 두 번째 인수로 전달받은 노드는 반드시 insertBefore 메서드를 호출한 노드의 자식 노드이어야 한다. 그렇지 않으면 DOMException 에러가 발생한다.
- 두 번째 인수로 전달받은 노드가 null이면 마지막 자식 노드로 추가된다.

### 39.6.6 노드 이동

DOM에 이미 존재하는 노드를 `appendChild` 또는 `insertBefore` 메서드를 사용하여 DOM에 다시 추가하면, 현재 위치에서 노드를 제거하고 새로운 위치에 노드를 추가한다.

### 39.6.7 노드 복사

`Node.prototype.cloneNode([deep: true | false])` 메서드를 사용해 노드를 복사할 수 있다.
1. deep에 true를 전달하면 깊은 복사를 하여 모든 자식 노드가 포함된 사본을 생성한다.
2. deep에 false를 전달하면 얕은 복사를 하여 노드 자신만의 사본을 생성한다.

### 39.6.8 노드 교체

`Node.prototype.replaceChild(newChild, oldChild)` 메서드의 newChild에는 교체할 새로운 노드를 전달하고, oldChild에는 교체될 노드를 전달한다.
1. oldChild는 replaceChild를 호출한 노드의 자식 노드이어야 한다.
2. oldChild는 교체된 후 DOM에서 제거된다.

### 39.6.9 노드 삭제

`Node.prototype.removeChild(child)` 메서드는 전달받은 child 노드를 DOM에서 삭제한다. child 노드는 메서드를 호출한 노드의 자식 노드이어야 한다.
## 39.7 어트리뷰트

### 39.7.1 어트리뷰트 노드와 attributes 프로퍼티

HTML 요소의 어트리뷰트는 어트리뷰트 노드로 변환되어 요소 노드의 `attributes` 프로퍼티에 저장된다.

`Element.prototype.attributes` 프로퍼티는 getter만 존재하는 읽기 전용 접근자 프로퍼티로, 요소 노드의 모든 어트리뷰트 노드의 참조가 담긴 `NamedNodeMap` 객체를 반환한다.

### 39.7.2 HTML 어트리뷰트 조작

`attributes` 프로퍼티를 통하지 않고 요소 노드에서 직접 어트리뷰트를 조작할 수 있다.

- `Element.prototype.getAttribute(attributeName)`: 어트리뷰트 값을 참조한다.
- `Element.prototype.setAttribute(attributeName, attributeValue)`: 어트리뷰트 값을 변경한다.
- `Element.prototype.hasAttribute(attributeName)`: 어트리뷰트 존재 여부를 확인한다.
- `Element.prototype.removeAttribute(attributeName)`: 어트리뷰트를 삭제한다.

### 39.7.3 HTML 어트리뷰트 vs. DOM 프로퍼티

HTML 어트리뷰트의 역할은 HTML 요소의 **초기 상태**를 지정하는 것이다. HTML 어트리뷰트 값은 초기 상태를 의미하며 변하지 않는다.

DOM 프로퍼티는 사용자의 입력에 의해 변경되는 **최신 상태**를 관리한다.

| 구분 | 역할 | 값 |
|---|---|---|
| HTML 어트리뷰트 | 초기 상태 지정 | 변하지 않는다 |
| DOM 프로퍼티 | 최신 상태 관리 | 변한다 |

단, `id` 어트리뷰트와 `id` DOM 프로퍼티처럼 사용자 입력과 관계없는 어트리뷰트는 어트리뷰트 노드와 DOM 프로퍼티가 항상 동일한 값을 유지한다.

### 39.7.4 data 어트리뷰트와 dataset 프로퍼티

`data` 어트리뷰트와 `dataset` 프로퍼티를 사용하면 HTML 요소에 정의한 사용자 정의 어트리뷰트와 자바스크립트 간에 데이터를 교환할 수 있다.

- `data` 어트리뷰트는 `data-` 접두사 다음에 임의의 이름을 붙여 사용한다. (예: `data-user-id`)
- `HTMLElement.dataset` 프로퍼티는 `DOMStringMap` 객체를 반환한다.
- `dataset` 프로퍼티는 `data-` 이후의 이름을 카멜 케이스로 변환한 프로퍼티를 가진다. (예: `data-user-id` → `dataset.userId`)
- `dataset` 프로퍼티에 값을 할당하면 HTML 요소에 `data` 어트리뷰트가 추가된다.

## 39.8 스타일

### 39.8.1 인라인 스타일 조작

`HTMLElement.prototype.style` 프로퍼티는 setter와 getter 모두 존재하는 접근자 프로퍼티로, 요소 노드의 인라인 스타일을 취득하거나 추가 또는 변경한다.

- `style` 프로퍼티를 참조하면 `CSSStyleDeclaration` 타입의 객체를 반환한다.
- CSS 프로퍼티는 케밥 케이스를 따르지만, `CSSStyleDeclaration` 객체의 프로퍼티는 카멜 케이스를 따른다. (예: `background-color` → `backgroundColor`)
- 케밥 케이스 그대로 사용하려면 대괄호 표기법을 사용한다. (예: `element.style['background-color']`)

### 39.8.2 클래스 조작

`class` 어트리뷰트에 대응하는 DOM 프로퍼티는 `className`과 `classList`이다.

**className**
- `Element.prototype.className` 프로퍼티는 `class` 어트리뷰트 값을 문자열로 반환한다.
- 문자열을 반환하므로 여러 클래스가 있을 경우 직접 파싱해야 하는 불편함이 있다.

**classList**
- `Element.prototype.classList` 프로퍼티는 `class` 어트리뷰트 값을 담은 `DOMTokenList` 객체를 반환한다.
- 유용한 메서드를 제공한다.

| 메서드 | 설명 |
|---|---|
| `add(...className)` | 인수로 전달한 클래스를 추가한다 |
| `remove(...className)` | 인수로 전달한 클래스를 삭제한다 |
| `item(index)` | index에 해당하는 클래스를 반환한다 |
| `contains(className)` | 해당 클래스가 포함되어 있는지 확인한다 |
| `replace(oldClassName, newClassName)` | 클래스를 교체한다 |
| `toggle(className)` | 클래스가 있으면 삭제하고, 없으면 추가한다 |

### 39.8.3 요소에 적용되어 있는 CSS 스타일 참조

`style` 프로퍼티는 인라인 스타일만 반환하므로, 클래스나 상속으로 적용된 스타일은 참조할 수 없다.

`window.getComputedStyle(element[, pseudo])` 메서드를 사용하면 요소에 적용된 모든 CSS 스타일을 참조할 수 있다.

- 인수로 전달한 요소 노드에 적용된 평가된 스타일을 `CSSStyleDeclaration` 객체에 담아 반환한다.
- 두 번째 인수로 `:before`, `:after` 같은 의사 요소를 지정할 수 있다.
